# 第2章｜MonoBehaviourとC#最低限

## 2-1. Unityスクリプトの正体

### MonoBehaviourとは何か
*   Unityのスクリプトは、基本的に `MonoBehaviour` というクラスを継承（ベースに）しています。
*   これのおかげで、Unityのゲームループ（毎フレームの更新など）に自動的に組み込まれます。
*   「魔法の土台」と考えてください。これを継承しないと、GameObjectに貼り付けることができません。

### Awake / Start / Update
*   **Awake**: ゲーム開始時（またはオブジェクト生成時）、最初に**1回だけ**呼ばれる。初期設定（自分自身の準備）に使います。
*   **Start**: Awakeの後、最初のフレームが始まる前に**1回だけ**呼ばれる。他のオブジェクトとの連携の準備に使います。
*   **Update**: **毎フレーム**呼ばれ続ける。入力の検知、移動処理、タイマーなど、「動き続けるもの」はここに書きます。

## 2-2. AIコードを読むためのC#

AIにコードを書かせた時、中身がブラックボックスだと修正や調整ができません。
「書けなくてもいいから、読める」レベルを目指します。

### class (クラス)
*   設計図のこと。ファイル名とクラス名は一致させるのがUnityのルールです。

### public / private / SerializeField
*   **public**: 外部（他のスクリプト）からアクセス可能。UnityエディタのInspectorにも表示される。
*   **private**: 内部専用。外部からは見えない。
*   **[SerializeField] private**: 内部専用だけど、UnityエディタのInspectorからは設定したい時に使う。**Unity開発ではこれが推奨されることが多いです。**

### if / for / 関数
*   **if (条件)**: 「もし〜なら」。HPが0なら死ぬ、キーが押されたらジャンプする。
*   **for / foreach**: 繰り返し。敵全員にダメージを与える、アイテムリストを表示する。
*   **関数 (Method)**: 処理のまとまり。`void Jump() { ... }` のように名前をつけて、後で呼び出せるようにしたもの。

## 📌 目標
👉 **AIが書いたスクリプトを「怖がらず読める」**
*   「ここは変数を宣言しているな」「ここはUpdateで毎回動いているな」と構造が見えればOKです。
*   詳細な文法は覚えなくていいです。わからなければAIに「この行は何をしているの？」と聞けばいいのです。
