# 第6章｜レンダーパイプライン基礎（判断力）

## 6-1. レンダーパイプラインとは

### 見た目と重さを決める心臓
*   Unityが3Dモデルやエフェクトをどうやって画面に描画するかを決める一連の処理ルールです。
*   画質（リッチさ）とパフォーマンス（軽さ）のトレードオフを決める重要な設定です。

## 6-2. Built-in / URP / HDRPの違い

### Built-in Render Pipeline (従来版)
*   昔からある標準。汎用性は高いが、設計が古く最適化が難しい。今から新規開発するなら選ぶ理由はあまりない。

### URP (Universal Render Pipeline)
*   **★AI時代の標準推奨**
*   モバイルからPC、Webまで幅広く対応。
*   軽量でカスタマイズ性が高く、画質も十分綺麗。
*   Shader GraphやVFX Graphなどの最新ツールが使える。

### HDRP (High Definition Render Pipeline)
*   PS5やハイエンドPC向け。実写レベルの超高画質。
*   **選ばない理由**: 非常に重い。WebGLやスマホではまともに動かないことが多い。個人開発やプロトタイプにはオーバースペック。

## 6-3. URP最低限設定

### Render Scale
*   解像度の調整。1.0がネイティブ。0.8などに下げると、見た目は少しボヤけるが劇的に軽くなる。
*   重い時の「緊急脱出ボタン」として覚えておきましょう。

### Shadow Distance (影の描画距離)
*   影の計算は非常に重い処理です。
*   「50m先にある小さな石の影」まで計算する必要はありません。これを短く設定するだけで軽くなります。

### Light設定の考え方
*   **Realtime Light**: 毎フレーム計算する光。綺麗だが重い。動くキャラやライトに使用。
*   **Baked Light**: 事前に計算してテクスチャに焼き付ける光。動かないが超軽量。背景や建物に使用。
*   これらを使い分けるのが「軽量化」のコツです。

📌 **目的**
👉 **「なぜ重いか」を説明できる**
*   「なんか重い」ではなく、「影の距離が無駄に長いから」「HDRPを選んでしまったから」と理由を特定できる判断力を養います。
