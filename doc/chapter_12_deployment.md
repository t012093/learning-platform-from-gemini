# 第12章｜ゲームデプロイ（ゴール）

## 12-1. ビルド基礎

### Windows / macOS
*   **Build Settings** からプラットフォームを選んでビルドします。
*   実行ファイル（.exe / .app）だけでなく、付随するデータフォルダもセットで配布する必要があります。

### WebGL
*   ブラウザで動く形式にビルドします。
*   **Compression Format**: Brotli や Gzip を選ぶとファイルサイズが小さくなりますが、サーバー側の設定が必要な場合があります。
*   **Color Space**: Linear か Gamma かで色味が変わるので注意が必要です。

### 解像度・品質設定
*   **Project Settings > Player** でアイコンや会社名を設定します。
*   これがデフォルトのままだと「作った感」が出ないので、必ず設定しましょう。

## 12-2. 配布と公開

### zip配布
*   PC向けの場合、ビルドされたファイルをzip圧縮して渡します。
*   Google DriveやDropboxで共有するのが一番手軽です。

### unityroom / Itch.io
*   WebGLの場合、**unityroom**（日本で人気）や**Itch.io**（世界的に人気）といった投稿サイトにアップロードするのがおすすめです。
*   サーバーを用意しなくても、zipをアップするだけでブラウザゲームとして公開できます。

### バージョン管理
*   「v1.0」「v1.1」のようにバージョンをつけます。
*   「バグ修正版」「新機能追加版」など、何が変わったかを記録します。

## 12-3. 振り返り

### AIがやったこと / 人間が判断したこと
*   完成後、「どこをAIに書いてもらったか」「どこで自分が苦労して判断したか」を言語化します。
*   それがあなたの「エンジニア/クリエイターとしての実績」になります。

### 次に改善する点
*   「もっと軽くできた」「コードが汚くなった」。
*   その反省は次のプロジェクトの「初期設計」に活かされます。
*   このループを回すことでしか、設計力は身につきません。
