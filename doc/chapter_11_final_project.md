# 第11章｜最終制作（ここが本番）

## 11-1. 企画

### 体験の核 (Core Mechanic)
*   「何をするゲームか」を1行で言えるようにします。
*   「ジャンプして避ける」「撃って壊す」「集めて育てる」。
*   あれもこれも詰め込まず、**たった1つの面白いこと**に集中します。

### 技術制約
*   「オープンワールドを作る！」 → 無理です。
*   「スマホとPC両対応！」 → 操作が大変です。
*   期間とスキルに見合った、実現可能な制約を自分で設定します。

### AIに任せる範囲
*   「プレイヤーの挙動」「敵のAI」「スコア計算」はAIに任せる。
*   「レベルデザイン（敵の配置）」「UIの見た目」「演出のタイミング」は人間がこだわる。
*   この線引きが完成度を左右します。

## 11-2. 実装

### プロンプト → 実装 → 修正 のループ
*   いきなり100点のコードは出ません。
*   60点のコードを出させて、実行してエラーを見て、エラー文をAIに投げて、80点にする。
*   このイテレーション（反復）の速度が開発スピードです。

### 設計を壊さない
*   AIは前の文脈を忘れて、突然設計を無視したコードを出すことがあります（シングルトンをやめる、変数をpublicにするなど）。
*   「前の設計方針を守って」と都度釘を刺すことが重要です。

## 11-3. テスト

### 初回起動
*   初めてプレイする人は何も知りません。「操作説明なしで遊べるか？」を確認します。

### 低スペPC / モバイル確認
*   自分のハイスペックPCで動いても、ユーザーの環境で動くとは限りません。
*   フレームレートが落ちていないかチェックします。

### エラー対応
*   プレイ中に「赤い文字（例外エラー）」が出ないか。
*   エラーが出るとゲームが止まったり、進行不能になったりします。
