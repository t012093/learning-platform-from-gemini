# 第0章｜AI時代のゲーム開発思想

## 0-1. AI時代に「書ける人」が減る理由

### AIが得意なこと / 苦手なこと
*   **得意**: 定型的なコード生成、既存パターンの実装、エラーの修正案提示、単純作業の高速化。
*   **苦手**: 目的の定義、「面白い」かどうかの判断、プロジェクト全体の整合性維持、長期的なメンテナンス性の担保、未知のバグへの創造的な対処。

### 人間の役割は「判断」と「構造」
*   これからのクリエイターに求められるのは、コードを1から書く力よりも、AIが生成したものが「目的に合っているか」を**判断**する力です。
*   また、AIが迷わないように、ゲーム全体の**構造（アーキテクチャ）**を設計し、適切な指示を与える「指揮官」としての役割が重要になります。

## 0-2. バイブコーディングとは何か

### 完璧な設計より“動く仮説”
*   「バイブコーディング」とは、細部にとらわれず、AIを使って「とりあえず動くもの」を高速で作るスタイルです。
*   頭で悩み続けるよりも、動くものを触って「なんか違う」「もっとこうしたい」という**バイブス（直感）**を頼りに開発を進めます。

### 失敗ログも資産
*   AIとの対話ログ、エラーが出たときの解決プロセス、これら全てが自分の知識資産となります。
*   「一発で成功させよう」と思わず、AIと一緒に試行錯誤するプロセスそのものを楽しみましょう。

## 📌 ここで教えること
👉 **「AIは部下、設計は人間」**
*   あなたが上司（ディレクター/アーキテクト）、AIは優秀な部下（プログラマー）です。
*   部下が働きやすい環境（明確な指示、良い設計）を作るのがあなたの仕事です。
