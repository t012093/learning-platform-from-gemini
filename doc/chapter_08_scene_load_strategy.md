# 第8章｜シーン分割とロード戦略

## 8-1. シーン構成設計

### Boot / Title / Lobby / Game
*   **Boot**: アプリ起動用。ロゴ表示や初期データのロードだけ行う。一瞬で終わる軽いシーン。
*   **Title**: スタートボタンがある画面。
*   **Lobby**: ステージ選択や装備変更。
*   **Game**: 実際のプレイ画面。
*   このように分割することで、メモリ管理がしやすくなり、チーム開発時の衝突も減ります。

### 常駐オブジェクトとは
*   `DontDestroyOnLoad` によって、シーンが切り替わっても消えずに残り続けるオブジェクト。
*   BGM管理や、プレイヤーの通算スコア管理などに使います。
*   使いすぎるとメモリを圧迫し続けるので注意が必要です。

## 8-2. 後読みの考え方

### Addressables（概念）
*   Unityの標準機能である「Resources」フォルダは、アプリ起動時に全部読み込まれるため、起動が遅くなる原因になります。
*   **Addressables (アドレッサブル)** は、必要な時に必要なものだけをインターネットやディスクから読み込むための仕組みです。
*   Webゲームでは「最初のロードを短くして、ゲーム開始までの離脱を防ぐ」ために必須の技術です。

### なぜ全部最初に読まないか
*   スマホゲームやWebゲームでは、通信量（パケット）とメモリ制限がシビアだからです。
*   「ステージ1」を遊んでいる間に、裏で「ステージ2」のデータをダウンロードする、といった賢い挙動が求められます。
